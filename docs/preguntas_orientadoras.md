# Preguntas orientadoras para discusión o cierre

1. ¿Qué diferencia hay entre abrir un archivo suelto y abrir una carpeta completa de proyecto?
   - Abrir una carpeta completa permite ver todo el proyecto y navegar entre `data`, `src`, `reportes`, `docs`; abrir un archivo suelto solo muestra ese archivo.
2. ¿Por qué conviene usar terminal integrada dentro del entorno?
   - Porque pude ejecutar scripts como `python src/03_generar_reporte.py` sin cambiar de aplicación y ver el resultado directamente.
3. ¿Qué ventaja tiene un entorno virtual en Python?
   - Entendí que ayuda a manejar dependencias y evita que las librerías del sistema interfieran con el proyecto.
4. ¿Qué pasaría si se ejecuta un script sin instalar sus dependencias?
   - El script puede fallar o mostrar errores de importación; por eso es importante instalar primero `requirements.txt`.
5. ¿Por qué los archivos CSV grandes son útiles para una práctica técnica?
   - Porque permiten trabajar con muchos datos reales y ver resultados más representativos, como los 10,000 registros de leche.
6. ¿Qué diferencia hay entre dato faltante, dato sospechoso y dato válido?
   - Un dato faltante es cuando no hay valor, un dato sospechoso es cuando el valor no parece correcto, y un dato válido es el que se puede usar sin problema.
7. ¿Qué criterio técnico pesa más en una institución rural: funcionalidad, compatibilidad o costo? Justifica.
   - En rural creo que costo y compatibilidad pesan más, porque se necesita software gratuito y que funcione en el equipo disponible.
8. ¿Por qué la depuración sistemática es mejor que cambiar código al azar?
   - Porque me di cuenta de que así se identifica el error real y se evitan cambios que no resuelven el problema.
9. ¿Qué evidencias demuestran que el entorno sí fue funcional?
   - El archivo `reporte_tecnico_semana13.txt` generado, la salida del script y la estructura clara del proyecto.
10. ¿Qué mejoras propondrías para una segunda versión del laboratorio?
   - Incluir más capturas de terminal, documentar los errores corregidos y revisar la instalación de dependencias con más detalle.
